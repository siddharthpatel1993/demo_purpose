Git Remote
 

-> In Git, the term remote is concerned with the remote repository. 
-> A remote repository is stored on a code hosting service like an internal server, GitHub, Subversion and more.
-> The developers can perform many operations with the remoter server such as clone, push, pull and more.


Check your remote ->

-> To check the configuration of the remote server, run the git remote command.
-> The git remote command allows accessing the connection between remote and local.
-> If you want to see the original existance of your cloned repository, use the git remote command.

Example -> $ git remote

Note: The given command would provide the remote name as the origin. Origin is the default name of the remote server, which is given my git.


Git remote -v ->

-> Git remote supports a specific option -v to show URLs that Git has stored as a short name.
-> These short names are used during the reading and writing operation.
-> Here, -v stands for verbose. We can use --verbose in place of -v.

Example -> $ git remote -v or $ git remote --verbose


Git remote Add ->

-> When we fetch a repository implicitley, git adds a remote for the repository.
-> Also, we can explicitly add a remote for a repository.
-> We can add a remote as a short nickname or short name.

Example -> $ git remote add short_name remote_URL


Fetching and pulling ->

-> You can fetch and pull data from the remote repository.
-> The fetch and pull command goes out to that remote server, and fetch all the data from the remote project that you don't have yet.
-> These commands let us fetch the references to all the branches from the remote.

Example -> $ git fetch remote_URI

To clone the remote repository from your remote projects, run the below command:

Example -> $ git clone remote_URI

When we clone a repository, the remote repository is added by a default name "origin". So, mostly, the command is used as git fetch origin.

Note: The git fetch origin fetches the updates that have been made to the remote server since you cloned it. The git fetch command only downloads the data
to the local repository; it doesn't merge or modify the data until you don't operate. You have to merge it manually into your repository when you want. 

To pull the repository, run the below command:
Example -> $ git pull remote

Note: The git pull command automatically fetches and then merges the remote data into your current branch. Pulling is an easier and comfortable workflow
than fetching. Because the git clone command sets up your local master branch to track the remote master branch on the server your cloned.


Pushing to Remote Branch ->

-> If you want to share the your project, you have to push it upstream.
-> The git push command is used to share a project or send updates to the remote server.

Example -> $ git push remote branch

It is a special command-line utility that specifies the remote branch and directory. When you have multiple branches on a remote server, then this command
assists you to specify your main branch and repository.


Git Remove Remote ->

You can remove a remote connection from a repository.

Example -> $ git remote rm destionation or $ git remote remove destionation


Git Remove Rename ->
 
Git allows renaming the remote server name so that you can use a short name in place of the remote server name. 

Example -> $ git remote rename old_name new_name


Git Show remote ->

To see additional information about a particular remote, use the git remote command along with show sub-command. It is used as:

Example -> $ git remote show remote_name


Git change remote(changing a Remote's URL) ->

The git remote set command is used to change the URL of the repository. It changes an existing remote repository URL.

Example -> $ git remote set-url remote_name new_URL
