Git Fetch
 

1. The "git fetch" command is used to fetch the updates from remote-tracking branches. 
2. It downloads commits, objects and refs from another repository. It also fetches branches and tags from one or more repositories.
3. Additionally, we can get the updates that have been pushed to our remote branches to our local machine.
4. As we know, a branch is a variation of our repositories main code, so the remote-tracking branches are the branches that have been set up to pull and push from remote 
   repository.


Ways to fetch Git Repository-

Scenario 1. To Fetch the remote repository:-
We can fetch the complete repository with the help of fetch command from a repository URL like a pull command does.

Example -> git fetch repository_URL

Scenario 2. To fetch a specific branch:-
We can fetch a specific branch from a repository. It will only access the element from a specific branch. 

Example -> git fetch branch_url branch_name

Scenario 3. To fetch all the branches simultaneously:-
The git fetch command allows to fetch all branchs simultaneously from a remote repository.

Example -> git fetch -all 

Scenario 4. To synchronize the local repository:-
Suppose, your team member has added some new features to your remote repository. So, to add these updates to your local repository, use the git fetch command. It is used as 
follow.

Example -> git fetch origin

Note:
1. The git fetch can fetch from either a single named repository or URL or from several repositories at once.
2. It can be considered as the safe version of the git pull commands.
3. The git fetch downloads the remote content but not update your local repo's working state.
4. When no remote server is specified, by default, it will fetch the origin remote.
