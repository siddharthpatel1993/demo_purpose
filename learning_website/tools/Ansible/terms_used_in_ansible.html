Terms Used in Ansible


Ansible Server =>  The machine where ansible is installed and from which all tasks and playbook will be run.

Modules => Basically, a module is a command or set of similar commands meant to be executed on the client-side.

Task=> A task in a section that consists of a single procedure to be completed.

Role=> A way of organising tasks and related files to be later called in a playbook.

Fact=> Information fetched from the client system from the global variables with the gather-facts information.

Inventory=> File containing data about the ansible client servers.  

Play=> Execution of Playbook.

Handler=> Task which is called only if a notifier is present.

Notifier=> Section attributed to a task which calls a handler if the output is changed.

Playbooks=> It consist code in YAML format, which describes tasks to be executed.

Host=> Nodes, which are automated by ansible. 

There are three ways you can push the instruction on node.

1. Adhoc commands {single line command} (No idempotency)
2. Modules {Single Work} (It can be used with YAML)
3. Playbook {More than one module} (It will be written in YAML format)
